D3.js is a JavaScript library for manipulating documents based on data. D3.js是一个基于数据处理文档的JavaScript库。 D3 helps you bring data to life using HTML, SVG, and CSS. D3帮助您使用HTML，SVG和CSS将数据带入生活。 D3's emphasis on web standards gives you the full capabilities of modern browsers without tying yourself to a proprietary framework, combining powerful visualization components and a data-driven approach to DOM manipulation. D3强调网络标准，为您提供现代浏览器的全部功能，无需将自己绑定到专有框架，将强大的可视化组件和数据驱动的方法结合到DOM操作中。

# Introduction ＃介绍

Read more tutorials .阅读更多教程 。

D3 allows you to bind arbitrary data to a Document Object Model (DOM), and then apply data-driven transformations to the document. D3允许您将任意数据绑定到文档对象模型（DOM），然后将数据驱动的转换应用到文档。 For example, you can use D3 to generate an HTML table from an array of numbers.例如，您可以使用D3从数字数组中生成HTML表格。 Or, use the same data to create an interactive SVG bar chart with smooth transitions and interaction.或者，使用相同的数据创建具有平滑过渡和交互的交互式SVG条形图。

D3 is not a monolithic framework that seeks to provide every conceivable feature. D3不是一个旨在提供所有可能的功能的单一框架。 Instead, D3 solves the crux of the problem: efficient manipulation of documents based on data.相反，D3解决了问题的症结所在：有效地处理基于数据的文档。 This avoids proprietary representation and affords extraordinary flexibility, exposing the full capabilities of web standards such as HTML, SVG, and CSS.这避免了专有代理，并提供了非凡的灵活性，展现了诸如HTML，SVG和CSS等Web标准的全部功能。 With minimal overhead, D3 is extremely fast, supporting large datasets and dynamic behaviors for interaction and animation.以最小的开销，D3非常快速，支持大数据集和动态行为以进行交互和动画。 D3's functional style allows code reuse through a diverse collection of official and community-developed modules. D3的功能风格允许通过各种官方和社区开发的模块来重复使用代码。

# Selections ＃选择

Read more about selections .阅读更多关于选择 。

Modifying documents using the W3C DOM API is tedious: the method names are verbose, and the imperative approach requires manual iteration and bookkeeping of temporary state.使用W3C DOM API修改文档非常乏味：方法名称是冗长的，而命令式方法需要手动迭代和记录临时状态。 For example, to change the text color of paragraph elements:例如，要更改段落元素的文本颜色： 

var paragraphs = document.getElementsByTagName("p");
for (var i = 0; i < paragraphs.length; i++) {
  var paragraph = paragraphs.item(i);
  paragraph.style.setProperty("color", "white", null);
}

D3 employs a declarative approach, operating on arbitrary sets of nodes called selections . D3采用了一种声明式的方法，在称为选择的任意节点集上进行操作。 For example, you can rewrite the above loop as:例如，您可以将上述循环重写为： 

d3.selectAll("p").style("color", "white");

Yet, you can still manipulate individual nodes as needed:但是，您仍然可以根据需要操作单个节点： 

d3.select("body").style("background-color", "black");

Selectors are defined by the W3C Selectors API and supported natively by modern browsers.选择器由W3C选择器API定义，并由现代浏览器本地支持。 The above examples select nodes by tag name ( "p" and "body" , respectively).以上示例通过标签名称（分别为"p"和"body" ）选择节点。 Elements may be selected using a variety of predicates, including containment, attribute values, class and ID.可以使用各种谓词来选择元素，包括遏制，属性值，类和ID。

D3 provides numerous methods for mutating nodes: setting attributes or styles; D3提供了许多用于变异节点的方法：设置属性或样式; registering event listeners;注册事件监听器; adding, removing or sorting nodes;添加，删除或排序节点; and changing HTML or text content.并更改HTML或文本内容。 These suffice for the vast majority of needs.这些满足绝大多数需求。 Direct access to the underlying DOM is also possible, as each D3 selection is simply an array of nodes.直接访问底层DOM也是可能的，因为每个D3选择只是一个节点数组。

# Dynamic Properties ＃动态属性

Readers familiar with other DOM frameworks such as jQuery should immediately recognize similarities with D3.熟悉其他DOM框架（如jQuery）的读者应立即认识到与D3的相似之处。 Yet styles, attributes, and other properties can be specified as functions of data in D3, not just simple constants.然而，D3中的样式，属性和其他属性可以被指定为数据的函数 ，而不仅仅是简单的常量。 Despite their apparent simplicity, these functions can be surprisingly powerful;尽管它们显然简单，但这些功能可能出人意料地强大; the d3.geoPath function, for example, projects geographic coordinates into SVG path data .例如， d3.geoPath函数将地理坐标 投影到SVG 路径数据中 。 D3 provides many built-in reusable functions and function factories, such as graphical primitives for area, line and pie charts. D3提供了许多内置的可重用函数和函数工厂，例如区域，行和饼图的图形基元 。

For example, to randomly color paragraphs:例如，要为段落随机着色： 

d3.selectAll("p").style("color", function() {
  return "hsl(" + Math.random() * 360 + ",100%,50%)";
});

To alternate shades of gray for even and odd nodes:为偶数和奇数节点交替灰度阴影： 

d3.selectAll("p").style("color", function(d, i) {
  return i % 2 ? "#fff" : "#eee";
});

Computed properties often refer to bound data.计算属性通常指的是绑定数据。 Data is specified as an array of values, and each value is passed as the first argument ( d ) to selection functions.数据被指定为一个值数组，并且每个值都作为第一个参数（ d ）传递给选择函数。 With the default join-by-index, the first element in the data array is passed to the first node in the selection, the second element to the second node, and so on.使用默认的索引连接，数据数组中的第一个元素被传递到选择中的第一个节点，第二个元素传递到第二个节点，依此类推。 For example, if you bind an array of numbers to paragraph elements, you can use these numbers to compute dynamic font sizes:例如，如果将数组数组绑定到段落元素，则可以使用这些数字来计算动态字体大小： 

d3.selectAll("p")
  .data([4, 8, 15, 16, 23, 42])
    .style("font-size", function(d) { return d + "px"; });

Once the data has been bound to the document, you can omit the data operator;数据绑定到文档后，您可以省略data运算符; D3 will retrieve the previously-bound data. D3将检索先前绑定的数据。 This allows you to recompute properties without rebinding.这允许您重新计算属性而无需重新绑定。

# Enter and Exit ＃进入并退出

Read more about data joins .阅读有关数据连接的更多信息 。

Using D3's enter and exit selections, you can create new nodes for incoming data and remove outgoing nodes that are no longer needed.使用D3的进入和退出选择，您可以为传入数据创建新节点并删除不再需要的传出节点。

When data is bound to a selection, each element in the data array is paired with the corresponding node in the selection.当数据绑定到选择时，数据数组中的每个元素都与选择中的对应节点配对。 If there are fewer nodes than data, the extra data elements form the enter selection, which you can instantiate by appending to the enter selection.如果节点数量少于数据，则额外的数据元素将形成输入选择，您可以通过追加enter选择来实例化enter选择。 For example:例如： 

d3.select("body")
  .selectAll("p")
  .data([4, 8, 15, 16, 23, 42])
  .enter().append("p")
    .text(function(d) { return "I’m number " + d + "!"; });

Updating nodes are the default selection—the result of the data operator.更新节点是默认选择 - data运算符的结果。 Thus, if you forget about the enter and exit selections, you will automatically select only the elements for which there exists corresponding data.因此，如果您忘记了进入和退出选择，您将只会自动选择存在相应数据的元素。 A common pattern is to break the initial selection into three parts: the updating nodes to modify, the entering nodes to add, and the exiting nodes to remove.一种常见模式是将初始选择分为三部分：要更改的更新节点，要添加的输入节点以及要删除的现有节点。 

// Update…
var p = d3.select("body")
  .selectAll("p")
  .data([4, 8, 15, 16, 23, 42])
    .text(function(d) { return d; });

// Enter…
p.enter().append("p")
    .text(function(d) { return d; });

// Exit…
p.exit().remove();

By handling these three cases separately, you specify precisely which operations run on which nodes.通过分别处理这三种情况，您可以精确指定在哪些节点上运行哪些操作。 This improves performance and offers greater control over transitions.这可以提高性能并提供对转换的更好控制。 For example, with a bar chart you might initialize entering bars using the old scale, and then transition entering bars to the new scale along with the updating and exiting bars.例如，对于条形图，您可以使用旧比例初始化输入条，然后将输入条更新为新比例尺以及更新和退出条。

D3 lets you transform documents based on data; D3可让您根据数据转换文档; this includes both creating and destroying elements.这包括创建和销毁元素。 D3 allows you to change an existing document in response to user interaction, animation over time, or even asynchronous notification from a third-party. D3允许您更改现有文档以响应用户交互，随着时间推移的动画，甚至是来自第三方的异步通知。 A hybrid approach is even possible, where the document is initially rendered on the server, and updated on the client via D3.甚至可以采用混合方法，其中文档最初在服务器上呈现，并通过D3在客户端进行更新。

# Transformation, not Representation ＃转化，而不是表示

D3 does not introduce a new visual representation. D3没有引入新的视觉表现。 Unlike Processing or Protovis , D3's vocabulary of graphical marks comes directly from web standards: HTML, SVG, and CSS.与Processing或Protovis不同，D3的图形标记词汇直接来自Web标准：HTML，SVG和CSS。 For example, you can create SVG elements using D3 and style them with external stylesheets.例如，您可以使用D3创建SVG元素并使用外部样式表进行设置。 You can use composite filter effects, dashed strokes and clipping.您可以使用复合滤镜效果，虚线描边和剪裁。 If browser vendors introduce new features tomorrow, you'll be able to use them immediately—no toolkit update required.如果浏览器供应商明天推出新功能，您可以立即使用它们 - 无需更新工具包。 And, if you decide in the future to use a toolkit other than D3, you can take your knowledge of standards with you!而且，如果您将来决定使用D3以外的工具包，则可以随身携带您的标准知识！

Best of all, D3 is easy to debug using the browser's built-in element inspector: the nodes that you manipulate with D3 are exactly those that the browser understands natively.最重要的是，使用浏览器的内置元素检查器，D3很容易调试：您使用D3操作的节点恰好是浏览器本身理解的节点。

# Transitions ＃转场

D3's focus on transformation extends naturally to animated transitions. D3对转型的关注自然延伸到动画过渡。 Transitions gradually interpolate styles and attributes over time.随着时间的推移，转换逐渐插入样式和属性。 Tweening can be controlled via easing functions such as “elastic”, “cubic-in-out” and “linear”.可以通过缓解“弹性”，“立方体进出”和“线性”等功能来控制补间。 D3's interpolators support both primitives, such as numbers and numbers embedded within strings (font sizes, path data, etc. ), and compound values. D3的内插器支持两种基元，例如嵌入在字符串中的数字和数字（字体大小，路径数据等 ）和复合值。 You can even extend D3's interpolator registry to support complex properties and data structures.您甚至可以扩展D3的插入器注册表以支持复杂的属性和数据结构。

For example, to fade the background of the page to black:例如，要将页面的背景淡化为黑色： 

d3.select("body").transition()
    .style("background-color", "black");

Or, to resize circles in a symbol map with a staggered delay:或者，以交错延迟的方式调整符号映射中的圆圈大小： 

d3.selectAll("circle").transition()
    .duration(750)
    .delay(function(d, i) { return i * 10; })
    .attr("r", function(d) { return Math.sqrt(d * scale); });

By modifying only the attributes that actually change, D3 reduces overhead and allows greater graphical complexity at high frame rates.通过只修改实际改变的属性，D3减少了开销，并允许在高帧率时具有更高的图形复杂度。 D3 also allows sequencing of complex transitions via events. D3还允许通过事件对复杂转换进行排序。 And, you can still use CSS3 transitions;而且，你仍然可以使用CSS3转换; D3 does not replace the browser's toolbox, but exposes it in a way that is easier to use. D3并没有取代浏览器的工具箱，而是以一种更易于使用的方式公开它。
